# README как онбординг: счастливый путь вперёд, devlog из сноски в продукт

## Контекст
README был написан для читателя, который уже знает, что такое кит. Два дефекта — оба про
онбординг, не про прозу:

- **Счастливый путь закопан.** Между `install` и единственной фразой, которая запускает кит,
  стояли 30 строк песочницы (`CLAUDE_CONFIG_DIR` + `--scope project`, ортогональность
  механизмов, грабли с именем плагина). Секция появилась в #14 осознанно и была проверена
  живьём — знание верное, место неверное: рецепт «как попробовать, ничего не трогая» читают
  единицы, а платят за него все.
- **devlog подан как опция.** Внизу, под заголовком `Optional`, тремя абзацами инвентаря
  компонентов. Первая строка README при этом заявляла «A Claude Code plugin» — единственное
  число, — и всё, что читалось дальше про devlog, звучало довеском. Память проекта между
  сессиями подана как nice-to-have.

Читатель, отскочивший от README, до кита не доходит вообще — цена дефекта не в тексте.

## Изменения
- **Счастливый путь — три строки.** Две команды `/plugin` + фраза. Песочница свёрнута в
  `<details>`: сохранены оба механизма и почему нужны оба, платформенная развилка по кредам
  (Linux/Windows vs Keychain), откат `rm -rf`, грабля `claude-code-harness`, не `harness`.
  Ничего из #14 не потеряно — сменился только уровень раскрытия.
- **devlog — часть дефолтной установки.** Install ставит оба плагина; шапка репо переписана с
  «A Claude Code plugin» на «Two Claude Code plugins … and keep a memory of it across sessions».
  Секция `The devlog companion: your project's memory` поднята **выше** `The opinion` и ведёт с
  проблемы (холодная сессия заново обсуждает решённое), а не с компонентов. Рамка: харнесс
  атемпорален (что правда о проекте), devlog эпизодичен (что в нём произошло) — разные работы,
  и на второй сессии реально теряют время. Requirements (Python 3, платформы) — закрывающей
  строкой; escape hatch «ведёте changelog руками — пропустите» сохранён.
- **Две новых строки в `What you can say`:** `/devlog:devlog` (вызов скилла в README не был
  показан ни разу) и явный полный `/claude-code-harness:claude-code-harness`.
- **`Four modes` удалён** — пересказывал таблицу фраз абстрактными существительными.
  Bootstrap/Audit живут в таблице и в `The opinion` (переехал ниже how-to: сначала что делает,
  потом почему так устроен); Extend/Explain свернулись в строку таблицы. `What ships` стал
  ссылкой в `Learn more` — инвентарь путей репо это контрибьюторский взгляд.

## Проверка
- **Имя команды взято из проверенного факта, не из своего окружения.** Запрошенный пример был
  `/claude-code-harness Хочу настроить Харнес, друг` — голая форма резолвится **только** у
  мейнтейнера, потому что кит догфудится симлинком `~/.claude/skills/claude-code-harness` и
  грузится личным скиллом без namespace. У подписчика скилл приезжает под namespace плагина:
  DEMO.md фиксирует со стенда `claude-code-harness:claude-code-harness`, `devlog:devlog`. В
  README ушла полная форма. Это ровно класс #12/#13/#14 — артефакт авторского окружения,
  который читается как инструкция для пользователя; на этот раз пойман до публикации.
- **Якорь проверен, а не предположен**: `#the-devlog-companion-your-projects-memory` для
  заголовка `## The devlog companion: your project's memory` (GitHub снимает двоеточие и
  апостроф, пробелы → дефисы).
- **Ссылки существуют**: `plugins/harness/references/operator-playbook.md`, `plugins/harness/`,
  `plugins/devlog/` — проверены по дереву.
- **Состав devlog-секции сверен с плагином**, не восстановлен по памяти: `skills/devlog/SKILL.md`
  (name: devlog → `/devlog:devlog`), `bin/devlog-reindex`, `hooks/session-start-digest.sh`.
- Reindex-оракул: `rebuild-index.py .claude/devlog` — зелёный (slug вычислен реальной
  `slugify()`, не руками).
