# Указатель пережил ноту, которая его запрещает

## Контекст

Оператор назвал остаток, который сам же не тронул в v1.17.1, чтобы не расширять скоуп: строка
change-sizing в шаблоне `## Working style` заканчивается указателем `— .claude/docs/workflow.md`,
а MVH-нота её не дропает. В MVH-проекте (`CLAUDE.md` + `settings.json`, больше ничего)
`.claude/docs/` не создаётся никогда — указатель висит.

Ирония в том, что запрет уже написан, и написан этой же нотой: «rules pointing at files that don't
exist are noise (detect-then-prescribe)». Нота перечисляла, какие **строки** дропнуть — и в этой
рамке хвост выжившей строки не виден. Тот же класс, что #18: правило есть, поверхность из-под него
торчит.

**Почему нельзя было просто дропнуть строку.** Phase 7 (строка 538): «MVH-on-request projects: only
the plan-mode and change-sizing greps apply» — греп `size the change` в MVH обязателен. Дроп строки
завалил бы собственный гейт кита. Отсюда форма фикса: **duty остаётся, указатель уходит** — не
«дропнуть», а «обрезать хвост».

## Изменения

- **MVH-нота** (`bootstrap-checklist.md:125-129`) — одна фраза: change-sizing-строка остаётся
  (Phase 7 её грепает), но её хвостовой `.claude/docs/workflow.md` снимается. Формулировка явно
  называет причину, по которой строка переживает обрезку, — чтобы следующая сессия не «упростила»
  ноту обратно в дроп-лист.

## Затронутые файлы

- `plugins/harness/references/bootstrap-checklist.md` — MVH-нота после Phase 2 шаблона (+3 строки)
- `CHANGELOG.md`, оба `plugin.json` (1.17.2, локстеп)

Не тронуто: сам гейт Phase 7, `practice-baseline.md`, `project-docs/*` (per-file стампы),
Phase 2b delta-список (там `size the change` починен в 1.17.1).

## Проверка

Перебор всех вхождений `.claude/` в обоих шаблонных блоках (Working style 97-123 + Reference
materials 146-164) — это и есть популяция, где дефект может жить:

| вхождение | судьба в MVH |
|---|---|
| change-sizing (104) | **был дефект** → нота снимает указатель, строка остаётся |
| ladder-semantics (114) · doc-with-code (116) · continuity (118, 120) | дропаются нотой |
| Reference-materials `.claude/docs/` + `.claude/devlog/` (157-160) | дропаются нотой |
| `docs/ARCHITECTURE.md … .claude/rules/` (156) | защищено «(only those that exist)» |

После фикса MVH-шейп `CLAUDE.md` не несёт ни одного указателя на файл, которого эта форма не
создаёт. Дублей ноты нет: `SKILL.md:167` несёт только summary-строку про MVH без дроп-листа,
`audit-checklist.md` и `operator-playbook.md` упоминают MVH в другом контексте.
