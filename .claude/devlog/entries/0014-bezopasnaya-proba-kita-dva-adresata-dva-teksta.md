# Безопасная проба кита: два адресата, два текста

## Контекст
Установка кита ничего не пишет в `~/.claude` — но это утверждение, и у потенциального
пользователя нет причин принимать его на веру. Способа попробовать кит, ничего не трогая, в
README не было. Заявка была сформулирована как «добавить DEMO.md, чтоб пользователи могли
безопасно попробовать», но у документа обнаружился другой адресат.

`.claude/DEMO.md` лежал в дереве неотслеживаемым и написан для мейнтейнера: его рецепт ставит
плагин из **локального чекаута** (`~/PROJECTS/claude-code-harness`), а грабли — про
симлинк-догфуд, итерацию по правкам без bump и «`plugin details` врёт вне стенда». Пользователь,
скопировав рецепт, спотыкается на первой же команде: такого пути у него нет, README учит
`nikitaCodeSave/claude-code-harness`. То есть документ обещал безопасную пробу и не давал её —
ровно класс #12/#13: артефакт, который читается как доставка, не доставляя.

## Изменения
Развилка решена разделением адресатов, а не переписыванием: два разных читателя — два текста.

- **README → «Try it in a throwaway config first»** — ядро рецепта для пользователя:
  `CLAUDE_CONFIG_DIR` + `--scope project`, источник GitHub, откат одним `rm -rf`. Записано,
  почему нужны **оба** механизма (они ортогональны): `CLAUDE_CONFIG_DIR` уносит весь `~/.claude`
  целиком, а `--scope project` держит установку в settings демо-проекта — **никакой проектный
  файл не отключает глобальный `~/.claude/CLAUDE.md`**, поэтому одного `--scope` мало.
- **`.claude/DEMO.md` закоммичен как есть** — авторский стенд, с маркером адресата в шапке,
  отсылающим пользователя в README. Выученные грабли сохранены целиком: они дорого достались
  (`plugin details` вне стенда рапортует про симлинк-догфуд, что читается как баг упаковки и
  провоцирует охоту на несуществующий дефект).
- Строка про креды **разведена по платформам**: Linux/Windows держат `.credentials.json`
  внутри конфиг-каталога → уезжает вместе с `CLAUDE_CONFIG_DIR`, копия избавляет от перелогина;
  macOS держит их в Keychain → копировать нечего. Оригинал DEMO.md проверялся только на Linux и
  такого различия не нёс; README адресован всем.

## Затронутые файлы
- `README.md` — секция «Try it in a throwaway config first» внутри Install
- `.claude/DEMO.md` — взят под git; добавлен блок «Адресат — мейнтейнер, не пользователь»
- `.gitignore` — allowlist `!/.claude/DEMO.md` (правка приехала вместе с документом)
- `CHANGELOG.md` — [1.16.2] Added

## Проверка
- **Рецепт исполнен целиком, а не вычитан.** Свежий `CLAUDE_CONFIG_DIR` + пустой git-репо →
  `claude plugin marketplace add nikitaCodeSave/claude-code-harness --scope project` →
  `claude plugin install claude-code-harness --scope project`: обе команды прошли, в
  `.claude/settings.json` демо-проекта легли `extraKnownMarketplaces` + `enabledPlugins`,
  `claude plugin list` показал `Version: 1.16.1, Scope: project, Status: enabled`.
- **Заодно сквозная проверка релиза #12/#13:** подписчик получил именно 1.16.1, и в доставленном
  `bootstrap-checklist.md` мёртвых permission-правил — ноль. Релиз доехал не по факту пуша, а по
  содержимому у консюмера.
- **Флаги подтверждены по `--help`**, не по памяти: `marketplace add --scope <user|project|local>`,
  `install -s/--scope`.
- **Ложная оговорка не попала в текст.** Клон в пробе пошёл по `git@github.com:` — выглядело как
  «CLI требует SSH», и в README чуть не уехало предупреждение про ключи. Причина оказалась в
  локальном `url.git@github.com:.insteadof https://github.com/` операторской машины; у
  пользователя без такого правила — HTTPS. Проверка конфига до написания сэкономила враньё в
  публичном README.
