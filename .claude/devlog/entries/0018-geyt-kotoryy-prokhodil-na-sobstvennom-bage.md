# Гейт, который проходил на собственном баге

## Контекст

v1.17.0 отгружен полчаса назад. Внешняя рефутация диффа пришла **после** пуша (агент молчал,
пока я релизил) и принесла CRITICAL: греп `grep -ci continuity CLAUDE.md`, написанный ровно ради
ловли CLAUDE.md без continuity-duty, **проходит на этом самом файле**.

Причина — коллизия токена. Блок `## Reference materials`, который тот же шаблон предписывает
копировать дословно, уже заканчивал строку этим словом **до** дифа (v1.16.3, строка 136):

```
- .claude/docs/workflow.md — flow: session ritual, plan, verification ladder, continuity
```

Фикстура из pre-diff шаблона (Working style без duty + этот блок): `continuity` = 1 → гейт зелёный,
duty нет. Три старых токена коллизии не имеют — проверил, они встречаются в шаблоне только в своих
duty-строках; `continuity` единственный.

**Почему живой оракул это не поймал — и это главный урок.** Оба прогона v1.17.0 (library +
sustained) писали duty корректно, поэтому греп возвращал ≥1 **по правильной причине**. Гейт,
который проходит и когда артефакт верен, и когда сломан, — не гейт. A/B по *выходу* бутстрапа
проверяет генератор; он структурно не может проверить **сам детектор**. Для детектора нужен
негативный фикстур — артефакт, который он обязан завалить. Его в плане не было, и это дыра
методологии, а не невезение.

## Изменения

- **Phase 7 — якорный токен**: `grep -ciE '^#{0,4} *-? *\*{0,2}Continuity'` — слово как **лейбл в
  начале строки** (duty-буллет или заголовок `## Continuity`), чего pointer-строка не даёт.
  Рядом — комментарий, запрещающий «упростить» обратно и объясняющий обе ловушки.
- **audit-checklist** — тот же якорь. Голый греп делал новый гэп-чек слепым к своей же популяции:
  pre-1.17.0 проекты несут ту самую Reference-materials строку → `→ 0` не срабатывает никогда.
  Заявление «expect this on **every** project bootstrapped before v1.17.0» было фактически неверным
  → «common».
- **Phase 2b vs Phase 7 — примирены.** Нетронутый Phase 2b предписывал обрезать Working style до
  «project-specific deltas (plan-mode duty + verification-ladder lines)», а practice-baseline §6 —
  это ровно continuity, при том что project embed — **дефолт** Phase 2b. Сессия, честно выполнившая
  обе фазы, удаляла duty и потом падала на гейте. Список дельт теперь явный (plan-mode · ladder ·
  change-sizing · continuity · doc-with-code) и объясняет, почему они переживают обрезку: они —
  project-side write-through, а «baseline говорит это слоем выше» есть ровно union-of-layers,
  который Phase 7 и отвергает. Тот же незакрытый список в audit §4 обновлён. (`size the change`
  нёс этот дефект до v1.17.0 — починен заодно.)
- **Остаток Phase-5-презумпций** (v1.17.0 закрыл 3 из 5): Phase 7 «with F0 as their due date» и
  SKILL.md (greenfield-строка + проза Mode 1) обещали ledger, которого на несustained greenfield нет.
- **Строка `.claude/devlog/entries/`** сделана условной по карьеру — на проекте с disciplined
  commits она указывала на директорию, которой не будет.
- **Phase 5 item 1** получил greenfield-оговорку: на 0 файлов раннера нет, оракул — TBD под `F0`,
  а `scripts/init.sh` против угаданного стека это запрет на выдумку в форме скрипта.
- **«the full shape»** (Phase 0) определён; `~16-line cost` → `~25-line` (блок реально 25 строк).

## Проверка

**Детектор проверен негативным фикстуром — то, чего не хватало в v1.17.0.** Фикстура собрана из
pre-diff шаблона: Working style без duty + дословный Reference-materials блок.

| артефакт | ожидание | голый греп (1.17.0) | `closes with an episodic` (предложение рефутера) | якорный (1.17.1) |
|---|---|---|---|---|
| buggy-фикстура | **0** (ловить) | **1 — пропускал** | 0 | **0** ✓ |
| library-бутстрап | ≥1 | 3 | **0 — ломал** | 1 ✓ |
| sustained-бутстрап | ≥1 | 3 | **0 — ломал** | 1 ✓ |
| section-style CLAUDE.md | ≥1 | 3 | **0 — ломал** | 1 ✓ |

**Фикс рефутера отвергнут по замеру, не по вкусу**: сессии перефразируют duty (наблюдалось:
«closes with a `/devlog:devlog` entry»), поэтому фразовый токен даёт 0 на **всех трёх** верных
бутстрапах — он заменил бы false pass на false fail в каждом проекте. Находка рефутера верна,
предложенное им лечение было бы хуже болезни; это аргумент за «рефутер называет дефект, автор
проектирует фикс», а не за слепое исполнение рекомендаций.

Живой прогон на 1.17.1 (greenfield library, 0 файлов / 0 коммитов): якорный греп = 1, три старых
= 1/2/1, `features.json` отсутствует (Phase 5 корректно скипнут), стабы без ссылок на F0,
devlog-запись #1 есть, CLAUDE.md 73 строки.
